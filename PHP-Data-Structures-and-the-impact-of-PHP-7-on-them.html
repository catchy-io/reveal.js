Array: PHP's untruthfulness

PHP “Arrays” are everything but plain, simple, static, arrays!

An array typically looks like this:

PHP “Arrays”

  • are dynamic: 
  • $array[] = 42;
unset($array[1]);
array_shift($array);
  • can be iterated one way:
  • foreach ($array as $elem) {}
next($array);
  • or another! 
  • prev($array);

exclusively with O(1) operations!

To achieve this, the underlying implementation of PHP “Arrays” requires a Doubly Linked List:

Thanks to this, the PHP “Array” enables Queue, Stack and Deque operations.

PHP “Arrays” can also

  • be accessed with string keys: 
  • $array["forty-two"] = 42;
  • or integer keys: 
  • $array[42] = "forty-two";

To achieve this, the underlying implementation of PHP “Arrays” requires an Hash Table as well:

Thanks to this, the PHP “Array” enables Dictionary, Map, Set,... operations.

From the manual: http://php.net/manual/en/language.types.array.php:

“This type is optimized for several different uses; it can be treated as an array, list (vector), hash table (an implementation of a map), dictionary, collection, stack, queue, and probably more.”

Optimized for anything ≈ Optimized for nothing!

Optimized for anything ≈ Optimized for nothing!

Array: PHP compared to C

  • In C: 100 000 integers (using long on 64bits => 8 bytes) can be stored in 0.76 MiB.
  • In PHP 5:
    • it will take ≅ 13.97 MiB!
    • A variable (containing an integer) takes 48 bytes.
    • The overhead for every “array” entries is about 96 bytes.
  • In PHP 5 7:
    • it will take ≅ 13.97 MiB! 4 MiB!!
    • A variable (containing an integer) takes 48 16 bytes.
    • The overhead for every “array” entries is about 96 20 bytes.

Data Structures

Structs (or records, tuples,...)

Structs (or records, tuples,...)

  • A struct is a value containing other values which are typically accessed using a name.
  • Example:

    Person => firstName / lastName

    ComplexNumber => realPart / imaginaryPart

Structs — Using array

$person = [
    "firstName" => "Patrick",
    "lastName" => "Allaert",
];

Structs — Using a class

$person = new PersonStruct(
    "Patrick", "Allaert"
);

Structs — Using a class (implementation)

class PersonStruct {
    public $firstName;
    public $lastName;

    public function __construct($firstName, $lastName) {
        $this->firstName = $firstName;
        $this->lastName  = $lastName;
    }

    public function __set($key, $value) {
        // a. Do nothing
        // b. trigger_error()
        // c. Throws an exception
    }
}

Structs — Pros and Cons

Go for objects if you want:

  • type hinting or a rigid structure (more OO)
  • to gain in terms of memory usage over speed

Go for arrays if you want

  • speed, despite bigger memory usage

(true) Arrays

An array is a fixed size collection where elements are each identified by a numeric index.

Obviously, you can use the regular PHP array structure...

or give a try to the native SplFixedArray

using SplFixedArray

Creation:

$array = new SplFixedArray(3);

Add entries:

$array[0] = 1;
$array[1] = 2;
$array[2] = 3;
$array->offsetSet(1);
$array->offsetSet(2);
$array->offsetSet(3);

Access them:

$array[0]; // gives 1
$array[1]; // gives 2
$array[2]; // gives 3

using string (8-bit int)

Creation:

$array = str_repeat(chr(0), 1024 * 1024);

Add entries:

$array[0] = chr(1 & 0xff);
$array[1] = chr(2 & 0xff);
$array[2] = chr(3 & 0xff);

Access them:

ord($array[0]); // gives 1
ord($array[1]); // gives 2
ord($array[2]); // gives 3

Pros and Cons

Go for SplFixedArray:

  • if you know in advance how many elements you have
  • to reduce memory usage
  • for extra speed if you can't go for PHP 7 yet

Queues

Queues

  • A queue is an ordered collection respecting First In, First Out (FIFO) order.
  • Elements are inserted at one end and removed at the other.

Queues — Using array

You can use the regular PHP array structure...

Creation:

$queue = [];

Add entries:

$queue[] = 1;
$queue[] = 2;
$queue[] = 3;
array_push($queue, 1);
array_push($queue, 2);
array_push($queue, 3);

Access them:

array_shift($queue); // gives 1
array_shift($queue); // gives 2
array_shift($queue); // gives 3

Queues — Using SplQueue

Creation:

$queue = new SplQueue();

Add entries:

$queue[] = 1;
$queue[] = 2;
$queue[] = 3;
$queue->enqueue(1);
$queue->enqueue(2);
$queue->enqueue(3);

Access them:

$queue->dequeue(); // gives 1
$queue->dequeue(); // gives 2
$queue->dequeue(); // gives 3

Stacks

Stacks

  • A stack is an ordered collection respecting Last In, First Out (LIFO) order.
  • Elements are inserted and removed on the same end.

Stacks — Using array

You can use the regular PHP array structure...

Creation:

$stack = [];

Add entries:

$stack[] = 1;
$stack[] = 2;
$stack[] = 3;
array_push($stack, 1);
array_push($stack, 2);
array_push($stack, 3);

Access them:

array_pop($stack); // gives 3
array_pop($stack); // gives 2
array_pop($stack); // gives 1

Stacks — Using SplStack

Creation:

$stack = new SplStack();

Add entries:

$stack[] = 1;
$stack[] = 2;
$stack[] = 3;
$stack->push(1);
$stack->push(2);
$stack->push(3);

Access them:

$stack->pop(); // gives 3
$stack->pop(); // gives 2
$stack->pop(); // gives 1

Queue/Stack — benchmark

Using 104 arrays of 1000 elements

Queue/Stack — Pros and Cons

Go for SplQueue/SplStack if you need to reduce memory usage (while under PHP 5) or want to be explicit on the data structures you use.

If speed is your concern or if you already focus on PHP 7, stay away from them (until some additional improvements are performed)!

Sets

Sets

  • A set is a collection with no particular ordering especially suited for testing the membership of a value against a collection or to perform union/intersection/complement operations between them

Sets — Using array

Creation:

$set = [];

Add entries:

$set[] = 1;
$set[] = 2;
$set[] = 3;

Checking presence:

in_array(2, $set); // true
in_array(5, $set); // false

Other operations:

array_merge($set1, $set2); // union
array_intersect($set1, $set2); // intersection
array_diff($set1, $set2); // complement
True performance killers!

Sets — Membership mis-usages

if (in_array($value, ["val1", "val2", "val3"]))
{
    // ...
}
if ($value === "val1" || $value === "val2" || $value === "val3")))
{
    // ...
}
switch ($value)
{
    case "val1":
    case "val2":
    case "val3":
        // ...
}

Sets — Using array (simple types)

Creation:

$set = [];

Add entries:

$set[1] = true; // Any dummy value is good BUT NULL!
$set[2] = true;
$set[3] = true;

Testing membership:

isset($set[2]); // true
isset($set[5]); // false

Other operations:

$set1 + $set2; // union
array_intersect_key($set1, $set2); // intersection
array_diff_key($set1, $set2); // complement

Sets — Using array (objects)

Creation:

$set = [];

Add entries:

$set[spl_object_hash($object1)] = $object1; // Use the object!
$set[spl_object_hash($object2)] = $object2;
$set[spl_object_hash($object3)] = $object3;

Testing membership:

isset($set[2]); // true
isset($set[5]); // false

Other operations:

$set1 + $set2; // union
array_intersect_key($set1, $set2); // intersection
array_diff_key($set1, $set2); // complement

Sets — Using SplObjectStorage

Creation:

$set = new SplObjectStorage();

Add entries:

$set->attach($object1);
$set->attach($object2);
$set->attach($object3);
$set[$object1] = true;
$set[$object2] = true;
$set[$object3] = true;

Testing membership:

isset($set[$object2]); // true
isset($set[$object5]); // false

Other operations:

$set1->addAll($set2); // union
$set1->removeAllExcept($set2); // intersection
$set1->removeAll($set2); // complement

Sets — Using bitsets

Creation:

$set = 0;

$set |= E_ERROR;

$set & E_ERROR; // true

Sets — Using bitsets (example)

define("REMOVE_FILES", 1 << 0);
define("REMOVE_DIRS",  1 << 1);
define("REMOVE_LINKS", 1 << 2);
define("REMOVE_EXEC",  1 << 3);
define("REMOVE_ALL",   ~0); // Setting all bits

function remove($path, $options = REMOVE_ALL)
{
    if (~$options & REMOVE_FILES && is_file($path))
        return false;

    if (~$options & REMOVE_DIRS && is_dir($path))
        return false;

    if (~$options & REMOVE_LINKS && is_link($path))
        return false;

    if (~$options & REMOVE_EXEC && is_executable($path))
        return false;
    // ...
}

Sets — Using bitsets (example)

remove("/tmp/removeMe", REMOVE_FILES | REMOVE_LINKS);

Maps

Maps

A map is a collection of key/value pairs where all keys are unique.

Maps

Creation:

$map = [];

Add entries:

$map["ONE"] = 1;
$map["TWO"] = 2;
$map["THREE"] = 3;

Combining maps:

  • the most popular sluggish way: 
    array_merge($map1, $map2);
  • the fast one: 
    $map2 + $map1;

Heaps

  • A heap is a tree-based structure in which all elements are ordered with largest key at the top, and the smallest one as leafs (or the opposite).

Heaps — Using Spl(Min|Max)Heap

$heap = new SplMinHeap;
$heap->insert(30);
$heap->insert(20);
$heap->insert(25);
var_dump($heap->top());

/* int(20) */

foreach ($heap as $elem) {
    var_dump($elem);
}
/*
int(20)
int(25)
int(30)
*/

Heaps — Conclusions

  • MUCH faster than having to re-sort() an array at every insertion.
  • Regular PHP arrays perform better than SplHeaps if you don't require a collection to be sorted at every single step.
  • SplPriorityQueue is very similar, consider it is the same as SplHeap but where the sorting is made on a key rather than the value.

Bloom filters

  • A bloom filter is a space-efficient probabilistic data structure used to test whether an element is member of a set.
  • False positives are possible, but false negatives are not!

Bloom filters — Using bloomy

Creation:

$bloom = new BloomFilter(
    10000, // capacity
    0.001  // (optional) error rate
           // (optional) random seed
);

Add entries:

$bloom->add("An element");

Testing membership:

$bloom->has("An element"); // true for sure
$bloom->has("Foo"); // false, most probably

Sparse dynamic arrays — Using Judy

Use a Judy array as you would use a regular PHP array:

$sparseArray = new Judy(Judy::INT_TO_INT);

$sparseArray[125] = 17;
$sparseArray[312] = 18;
$sparseArray[859] = 25;

echo $sparseArray[125]; // will print 17
echo $sparseArray[42];  // will print 0

A memory-efficient bitset is also possible:

$bitset = new Judy(Judy::BITSET);
$bitset[124] = true;

print $bitset[124]; // will print 1
print $bitset[100]; // will print 0

$bitset[124] = false; // is the same as unset($bitset[124])

Conclusions

  • Use appropriate data structure. It will keep your code self-documenting and fast.
  • Think about the time and space complexity involved by your algorithms.
  • Name your variables accordingly: use “Map”, “Set”, “List”, “Queue”,... to describe them instead of (always) naming it: $array.
